## Executive Summary

LifeFlow already has the plumbing for AI via its BYOK architecture — encrypted API key storage, two stub endpoints. This spec proposes **42 AI features** generated by a 7-expert panel, synthesized into a phased roadmap. The guiding principles:

- **Privacy-first**: All AI calls use user's own API key. No telemetry. Local models supported.
- **Graceful degradation**: Every AI feature has a non-AI fallback. LifeFlow works fully without AI.
- **Contextual, not generic**: Because LifeFlow owns the full hierarchy (Area → Goal → Task → Subtask), AI has rich context that generic chatbots lack.
- **Competitive moat**: Self-hosted AI task intelligence is something Todoist, Things 3, and Notion can't offer without cloud lock-in.

## Panel Discussion

### 1. Life Coach — Dr. Maya Reeves

*"The most powerful AI isn't the one that does things for you — it's the one that asks the right question at the right time."*

#### Feature 1: **Weekly Review AI Copilot** ⭐ MUST-HAVE

**What:** During the existing weekly review flow, AI analyzes completed/missed tasks, habit streaks, and focus time — then generates reflective prompts and pattern observations.

**User Story:** *As a user completing my weekly review, I want AI-generated insights like "You completed 12 Work tasks but only 2 Health tasks — is that the balance you intended?" so I can course-correct before another week passes.*

**How it works:**
1. User opens Weekly Review (existing feature)
2. AI receives: tasks completed/created this week, habit logs, focus sessions, life area distribution, prior review reflections
3. Returns: 3-5 personalized reflection questions, a "pattern alert" if something is off, and a suggested focus for next week
4. User can accept, edit, or ignore each suggestion

**AI capabilities:** Text generation (GPT-4/Claude), context window for weekly data (~2K tokens)
**Privacy:** All data stays in the prompt → user's API key → response. Nothing stored externally.

---

#### Feature 2: **Goal Decomposition Assistant** ⭐ MUST-HAVE

**What:** Given a high-level goal (e.g., "Run a marathon by October"), AI generates a structured task breakdown with milestones, realistic timelines, and dependencies.

**User Story:** *As a user who just created a new goal, I want AI to suggest a realistic task breakdown with milestones so I don't have to figure out the entire path myself.*

**How it works:**
1. User creates goal, clicks "AI Decompose"
2. AI receives: goal title, description, due date, life area context, user's existing tasks/commitments
3. Returns: milestone suggestions, task list with estimated durations, suggested dependencies, recommended recurring check-ins
4. User reviews, edits, then batch-creates the accepted items

**AI capabilities:** Structured output generation (function calling / JSON mode), domain knowledge
**Privacy:** Goal context + existing load summary sent to AI. No PII beyond what user typed.

---

#### Feature 3: **Accountability Partner Bot**

**What:** Proactive AI that checks in during the day based on what you planned. Uses push notifications or in-app toasts.

**User Story:** *As a user who planned 5 tasks for today but completed 0 by 2pm, I want a gentle nudge like "Hey, you planned to start 'Interval training' this morning — want to reschedule or tackle it now?" so I stay accountable without needing a human coach.*

**How it works:**
1. Background scheduler (existing `scheduler.js`) triggers at configurable intervals
2. Compares planned today tasks vs. completion status
3. Generates contextual nudge (not generic "do your tasks!" — references specific task names and context)
4. Delivers via existing push notification system or in-app toast

**AI capabilities:** Short text generation, scheduling logic (mostly rule-based with AI-generated message text)
**Priority:** Nice-to-have

---

#### Feature 4: **Life Balance Radar**

**What:** AI generates a periodic "life balance score" across all life areas, visualized as a radar chart, with commentary on imbalances.

**User Story:** *As a user with 6 life areas, I want to see a radar chart showing my recent activity distribution with AI commentary like "Your 'Relationships' area has had zero activity in 3 weeks — consider scheduling a small step" so I maintain holistic well-being.*

**How it works:**
1. Weekly or on-demand, aggregate: tasks completed, focus time, habit logs per life area
2. AI generates commentary on the distribution, flags neglected areas, suggests micro-tasks
3. Rendered as SVG radar chart with AI annotations

**AI capabilities:** Data analysis + text generation. Could work with a smaller model or even rule-based scoring + AI commentary.
**Priority:** Nice-to-have

---

#### Feature 5: **Habit Formation Coach**

**What:** When a user creates a new habit, AI suggests an implementation strategy: cue-routine-reward structure, habit stacking suggestions (linked to existing habits), and predicted difficulty.

**User Story:** *As a user adding a new "Meditate 10 min" habit, I want AI to suggest "Stack this after your existing 'Morning coffee' routine — people who attach new habits to existing cues have 3x higher success rates" so I'm set up for success from day one.*

**AI capabilities:** Text generation with behavioral science knowledge, access to user's existing habits for stacking
**Priority:** Nice-to-have

---

#### Feature 6: **Motivational Pattern Recognition**

**What:** AI analyzes your productivity patterns over time and identifies what conditions correlate with high-output days vs. low-output days.

**User Story:** *As a user, I want to see insights like "Your most productive days are Tuesdays and Wednesdays, especially when you start with a focus session before 9am" so I can optimize my schedule around my natural rhythms.*

**AI capabilities:** Pattern analysis over time-series data (focus sessions, completions, habit logs). Could use embeddings or statistical analysis + LLM for narrative explanation.
**Priority:** Moonshot (needs 30+ days of data to be useful)

### 4. Fintech Product Manager — Alex Chen

*"The features that win aren't the smartest — they're the ones that reduce friction to zero. AI should eliminate steps, not add them."*

#### Feature 1: **Intelligent Task Capture (NLP v2)** ⭐ MUST-HAVE

**What:** Upgrade the existing NLP parser with AI-powered understanding. Current parser uses regex; AI version understands natural language, extracts entities, infers context.

**User Story:** *As a user typing "call dentist next tuesday for the annual checkup, high priority, health area", I want AI to extract: title="Call dentist for annual checkup", due_date=next Tuesday, priority=3, area=Health, and create the task in the right goal — not just pattern matching but understanding.*

**How it works:**
1. User types in quick capture (existing `N` shortcut)
2. If AI key is configured, send to AI with function-calling schema for task fields
3. AI returns structured JSON: `{title, due_date, due_time, priority, area_id, goal_id, tags, estimated_minutes, recurring}`
4. Fallback to existing regex parser if no AI key

**Comparison to existing:** Current NLP handles "#tag", "!priority", "tomorrow" keywords. AI version understands "remind me", "every other week", "for my marathon goal", "about 30 minutes".

**AI capabilities:** Function calling / structured output, entity extraction
**Priority:** Must-have (transforms daily interaction)

---

#### Feature 2: **Smart Task Routing** ⭐ MUST-HAVE

**What:** When a task is created (from inbox, quick capture, or API), AI suggests the best goal, tags, priority, and estimated time based on similar past tasks.

**User Story:** *As a user adding "Write quarterly report" to inbox, I want AI to suggest: Goal="Q2 Deliverables", Priority=High, Tags=[work, writing], Estimated=120min — because I created similar tasks before.*

**How it works:**
1. On task creation, query recent tasks for the user (last 100)
2. AI classifies the new task based on historical patterns
3. Suggestions shown as pre-filled fields (user confirms or changes)

**AI capabilities:** Classification + few-shot learning from user's own data
**Priority:** Must-have (friction reduction)

---

#### Feature 3: **API-First AI Gateway**

**What:** Expose the AI features as proper REST API endpoints so users can build integrations, shortcuts, and automations on top.

**User Story:** *As a power user, I want to call `POST /api/ai/decompose` with a goal description and get back structured tasks/milestones JSON, so I can build my own iOS Shortcut that creates goals from voice memos.*

**Proposed endpoints:**
```
POST /api/ai/capture      — NLP text → structured task
POST /api/ai/decompose    — Goal → tasks/milestones
POST /api/ai/classify     — Task → suggested goal/tags/priority
POST /api/ai/summarize    — Date range → narrative summary
POST /api/ai/review       — Weekly/daily review data → insights
POST /api/ai/schedule     — Task list → optimized schedule (existing stub)
POST /api/ai/suggest      — Task → subtask suggestions (existing stub)
```

**AI capabilities:** All of the above, wrapped in clean REST with consistent JSON schemas
**Priority:** Must-have (platform play)

---

#### Feature 4: **Automation Rule AI Builder**

**What:** Let users describe automation rules in natural language. AI converts to trigger/condition/action JSON.

**User Story:** *As a user, I want to type "When I complete all subtasks of a task, automatically mark it as done and send me a congratulations push notification" and have AI generate the correct automation rule configuration.*

**How it works:**
1. User types natural language description in automation builder
2. AI maps to existing trigger types (19), action types (19), and conditions
3. Returns structured rule config. User reviews and saves.

**AI capabilities:** Intent classification + structured output mapping to existing automation schema
**Priority:** Nice-to-have (power user feature)

---

#### Feature 5: **Cross-Entity Search with Semantic Understanding**

**What:** Upgrade the existing FTS5 search with semantic/embedding-based search. "Things I need to do before my trip" finds tasks tagged travel, packing lists, passport renewal — even without the word "trip".

**User Story:** *As a user searching "financial stuff I've been procrastinating on", I want to find my overdue finance tasks, stalled savings goals, and dismissed habit reminders — not just keyword matches.*

**How it works:**
1. Generate embeddings for all tasks/goals/notes (batch, on change)
2. Store in a local vector table (SQLite with a vector extension, or a simple cosine similarity table)
3. On search, embed the query, find nearest neighbors, rank by relevance + recency
4. Fallback to FTS5 if no AI key

**AI capabilities:** Embedding generation + similarity search
**Priority:** Nice-to-have (significant infrastructure investment)

---

#### Feature 6: **Suggested Automations from Behavior**

**What:** AI observes repeated manual patterns and suggests automations. "You've moved 12 tasks to 'doing' then set priority to high in the last month. Want to automate that?"

**User Story:** *As a user, I want AI to notice my repeated behaviors and suggest "It looks like you always tag tasks as 'urgent' when they're due within 2 days. Create an automation rule?"*

**AI capabilities:** Pattern mining over audit log / task history + structured suggestion generation
**Priority:** Nice-to-have

### 5. Behavioral Psychologist — Dr. Lena Okafor

*"Every feature you build is a choice architecture. AI gives us the power to design environments that make the right thing the easy thing."*

## Technical Architecture Overview

### AI Service Layer (Refactored `src/services/ai.js`)

```
src/services/ai/
  index.js              — Public API: suggest, schedule, capture, decompose, classify, etc.
  provider.js           — Provider abstraction (OpenAI, Anthropic, Ollama, Custom)
  prompts/
    capture.js          — NLP v2 prompt templates
    decompose.js        — Goal decomposition prompts
    daily-plan.js       — Intelligent daily planning prompts
    review.js           — Weekly/daily review copilot prompts
    classify.js         — Task routing classification prompts
    summarize.js        — Narrative summary prompts
  transparency.js       — Prompt logging, data minimization, pre-flight data
  embeddings.js         — Local embedding generation (optional)
  queue.js              — Offline AI request queue
```

### Database Additions

```sql
-- AI interaction log (encrypted at rest)
CREATE TABLE ai_interactions (
  id INTEGER PRIMARY KEY,
  user_id INTEGER REFERENCES users(id) ON DELETE CASCADE,
  feature TEXT NOT NULL,          -- 'capture', 'decompose', 'daily_plan', etc.
  provider TEXT NOT NULL,         -- 'openai', 'ollama', etc.
  prompt_hash TEXT,               -- SHA-256 of prompt (for dedup, not content)
  tokens_used INTEGER DEFAULT 0,
  accepted BOOLEAN DEFAULT 0,     -- did user accept the suggestion?
  created_at TEXT DEFAULT (datetime('now'))
);

-- Optional: local embeddings table
CREATE TABLE embeddings (
  id INTEGER PRIMARY KEY,
  entity_type TEXT NOT NULL,      -- 'task', 'goal', 'note'
  entity_id INTEGER NOT NULL,
  embedding BLOB NOT NULL,        -- float32 array as blob
  model TEXT NOT NULL,            -- 'all-MiniLM-L6-v2'
  updated_at TEXT DEFAULT (datetime('now')),
  UNIQUE(entity_type, entity_id)
);
```

### Settings Additions

```
ai_provider          — 'openai' | 'anthropic' | 'ollama' | 'custom'
ai_base_url          — Base URL for API calls
ai_model             — Model name (e.g., 'gpt-4', 'llama3:8b')
ai_api_key           — Encrypted API key (existing)
ai_transparency_mode — 'always' | 'trust' | 'off'
ai_data_minimization — 'strict' | 'standard' | 'full'

```

## Open Questions

1. **Embedding model size vs. accuracy:** all-MiniLM-L6-v2 is 80MB and decent. Larger models (e5-large) are 1.3GB but much better. Default to small?

2. **Streaming responses:** Should AI features stream responses (typewriter effect) or wait for complete response? Streaming feels more responsive but adds complexity.

3. **AI feature gating:** Should AI features be visually hidden when no API key is configured, or shown with a "Configure AI in Settings" prompt? Latter creates discovery but might frustrate users who don't want AI.

4. **Rate limiting for AI calls:** User's own API key has its own rate limits, but should LifeFlow add a safety throttle to prevent accidental cost spikes? (e.g., max 50 AI calls/hour)

5. **Multi-model routing:** For cost optimization, should simple features (classification) use a cheap/fast model while complex features (decomposition) use a capable model? Let user configure per-feature?

6. **Template marketplace hosting:** Self-hosted JSON sharing, or a lightweight community server? Privacy implications of each?

7. **Financial data sensitivity:** Should custom fields marked as "financial" get extra encryption or AI exclusion by default?
